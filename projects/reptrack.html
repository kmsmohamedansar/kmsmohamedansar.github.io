Overview

What it is

A disciplined training log—not a social feed or analytics dashboard pretending to be a product.

Problem: logging in the gym needs to be fast; reviewing last session needs to be obvious. Most apps add noise.

Response: RepTrack keeps a tight session model—workouts, exercises, sets—with UI that stays calm so the work stays in the gym.

What I built

Delivery scope

Four execution pillars—each maps to shippable surface area, not slideware.

01

Session flow

Workout lifecycle from create → exercise → sets, optimized for thumbs and time pressure.

02

Data layer

SwiftData models and relationships that stay understandable as features grow.

03

UI system

Consistent hierarchy, spacing, and native controls—no custom chrome for its own sake.

04

Store readiness

Metadata, assets, and submission path through App Store Connect under real constraints.
