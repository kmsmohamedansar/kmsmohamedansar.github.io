How I Built a Local Grocery Product Q&A Assistant Using Instacart Data and RAG

Introduction:
Fully local semantic search + Q&A over the Instacart Market Basket dataset. No APIs. FAISS for fast similarity search, a compact LLM (flan-t5-base) for answers, and a Gradio UI for interaction.

What it solves (in plain English):

  • ‚ÄúWhat‚Äôs a good gluten-free snack?‚Äù ‚Üí Finds semantically similar products across aisles/departments.
  • ‚ÄúShow breakfast items with dairy‚Äù ‚Üí Retrieves relevant items and explains why.
  • Works offline: perfect for demos, cost control, and privacy.

Pipeline Steps:

  1. Load CSVs (products, aisles, departments) and merge readable metadata.
  2. Create a combined_text description for each product.
  3. Embed with sentence-transformers/all-MiniLM-L6-v2.
  4. Index with FAISS; run millisecond top-k similarity search.
  5. Compose a grounded prompt and answer with flan-t5-base.
  6. Expose a simple Gradio chat for end users.

Design choices:

  • Local embeddings + FAISS for reproducibility and speed.
  • Small LLM to keep inference cheap and portable.
  • Deterministic preprocessing so results are easy to debug.

Core Code Highlights:

# Install (notebook)
!pip install -q langchain langchain-community faiss-cpu sentence-transformers transformers gradio kagglehub

# Data
import kagglehub, pandas as pd, os
data_path = kagglehub.dataset_download("yasserh/instacart-online-grocery-basket-analysis-dataset")
products = pd.read_csv(os.path.join(data_path, "products.csv"))
aisles = pd.read_csv(os.path.join(data_path, "aisles.csv"))
depts = pd.read_csv(os.path.join(data_path, "departments.csv"))
df = products.merge(aisles, on="aisle_id").merge(depts, on="department_id")
df["combined_text"] = df.apply(lambda r: f"Product: {r['product_name']}. Aisle: {r['aisle']}. Department: {r['department']}.", axis=1)

# Embeddings + Index
from langchain.embeddings import HuggingFaceEmbeddings
from langchain.vectorstores import FAISS
emb = HuggingFaceEmbeddings(model_name="sentence-transformers/all-MiniLM-L6-v2")
store = FAISS.from_texts(df["combined_text"].tolist(), emb)

# Local LLM
from transformers import pipeline
gen = pipeline("text2text-generation", model="google/flan-t5-base", max_length=256)

def answer(q):
    ctx = "\n".join([d.page_content for d in store.similarity_search(q, k=4)])
    prompt = f"Use the context to answer briefly.\nContext:\n{ctx}\n\nQuestion: {q}"
    return gen(prompt)[0]["generated_text"]

# Gradio
import gradio as gr
ui = gr.Interface(fn=answer, inputs="text", outputs="text", title="Grocery AI (Local RAG)")
ui.launch()

Metrics & Checks: Spot-checked retrieval quality (top-k relevance), latency < 100ms for FAISS lookups on laptop, answer coherence via qualitative prompts.

Business Impact: Demonstrates how to stand up a private RAG system for product catalogs‚Äîuseful for retail search, merchandising, and support assistants without cloud cost.

üì∞ Yahoo Finance News Scraper ‚Äî Selenium + BeautifulSoup

Headless Chrome scraper for JavaScript-rendered news: headlines, URLs, dates, authors, full article text to CSV/DataFrame.

Tech: Python, Selenium (Headless Chrome), BeautifulSoup, Pandas, ChromeDriver

Project Overview

  • Problem: Yahoo Finance loads much of its content via JavaScript; static scrapers miss it.
  • Solution: Use Selenium to render the page (headless Chrome), then parse with BeautifulSoup.
  • Output: Clean DataFrame with headline, URL, author, timestamp, and article text; export to CSV.

How It Works

  1. Spin up headless Chrome via Selenium; load https://finance.yahoo.com/ and wait for JS to render.
  2. Extract homepage headlines/links from <h3> ‚Üí <a>.
  3. Visit top N articles; capture <time>, author hints, and content blocks (e.g., .caas-body).
  4. Assemble into a Pandas DataFrame and save to yahoo_finance_news_detailed.csv.

Core Snippet

from selenium import webdriver
from selenium.webdriver.chrome.options import Options
from bs4 import BeautifulSoup
import pandas as pd, time

opts = Options()
opts.add_argument("--headless"); opts.add_argument("--disable-dev-shm-usage"); opts.add_argument("--no-sandbox")
driver = webdriver.Chrome(options=opts)

driver.get("https://finance.yahoo.com/"); time.sleep(5)
soup = BeautifulSoup(driver.page_source, "html.parser")

news_links = []
for h3 in soup.find_all("h3"):
    a = h3.find("a")
    if a and a.text.strip():
        href = a["href"]
        if href.startswith("/"): href = "https://finance.yahoo.com" + href
        news_links.append({"headline": a.text.strip(), "url": href})

rows = []
for art in news_links[:5]:
    driver.get(art["url"]); time.sleep(3)
    s = BeautifulSoup(driver.page_source, "html.parser")
    headline = s.find("h1").get_text(strip=True) if s.find("h1") else ""
    dt_tag = s.find("time"); dt = dt_tag.get("datetime") if dt_tag else ""
    content_div = s.find("div", class_="caas-body")
    content = content_div.get_text(" ", strip=True) if content_div else " ".join(p.get_text(strip=True) for p in s.find_all("p"))
    rows.append({**art, "date": dt, "content": content})

pd.DataFrame(rows).to_csv("yahoo_finance_news_detailed.csv", index=False)

Extensions: schedule runs, sentiment analysis for market signals, or feed a dashboard.

Impact: Automates JS-heavy news collection reliably‚Äîuseful for alerts, trend tracking, or downstream NLP.

üß† Smart Product Categorization ‚Äî Zero-Shot NLP

Automatic product-to-category matching for e-commerce using zero-shot learning and simple rules.

Tech: Python, pandas, ast, Transformers (facebook/bart-large-mnli), Zero-Shot Learning

Smart Product Categorization with NLP and Zero-Shot Learning

Problem: Large catalogs get miscategorized; manual QA is slow and costly. We need a scalable way to verify (and improve) product‚Üícategory mapping.

Data: Walmart product dataset with product names and category paths.

Approach:

  1. Parse categories: Convert category-path strings ‚Üí Python lists; extract the last (most specific) category.
  2. Simple rule: Case-insensitive keyword inclusion check (quick baseline).
  3. Zero-shot check: Use facebook/bart-large-mnli to score how well the product name fits the last category (0‚Äì1 confidence).
  4. Review: Low scores ‚Üí likely miscategorized ‚Üí flag for correction.

Core Code Highlights:

import pandas as pd, ast
df = pd.read_csv("/content/walmart-products.csv")

def parse_categories(cat_str):
    try: return ast.literal_eval(cat_str)
    except: return []

df["categories_list"] = df["categories"].apply(parse_categories)
df["last_category"] = df["categories_list"].apply(lambda x: x[-1] if x else None)

def keyword_match(row):
    if not row["last_category"]: return False
    return row["last_category"].lower().replace(" ","") in row["product_name"].lower().replace(" ","")

df["simple_match"] = df.apply(keyword_match, axis=1)

# Zero-shot classification
!pip -q install transformers
from transformers import pipeline
zshot = pipeline("zero-shot-classification", model="facebook/bart-large-mnli")

def match_score(row):
    if not row["last_category"] or not isinstance(row["product_name"], str):
        return None
    res = zshot(row["product_name"], row["last_category"])
    return res["scores"][0]

df["category_match_score"] = df.apply(match_score, axis=1)
print(df[["product_name","last_category","category_match_score"]].head())

Performance Notes: Zero-shot on CPU is slow (minutes per batch). For scale: batch inputs, run on GPU, cache model, or pre-screen with keyword/embedding filters.

Impact: Flags subtle misclassifications that rules miss; improves findability and reduces catalog QA time.

Role heuristics (players):

  • CF: score = goals + penalties ‚àí expected goals

  • DM: score = assists + 2√ósuccessful tackles + 0.5√óinterceptions
  • LB/RB: score = 2√óinterceptions + 0.5√ótackles + duels won
  • CB: score = interceptions + 2√ótackles + 3√óclearances
  • GK: score = 2√ósaves + 3√óclean sheets
