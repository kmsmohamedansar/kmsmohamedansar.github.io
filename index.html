Problem: Toy examples skip the ops. I wanted a small project that behaves like production: idempotent steps, failure handling, and observability.

Solution: An orchestrated pipeline (Ingest → Validate → Featurize → Train → Serve) with exponential backoff retries, structured logs, and live demo via Gradio.

Architecture
  • Steps are pure functions with on-disk artifacts to enable re-runs and caching.

Results

What I learned
  • Small reliability features (retries, idempotency) save huge debugging time.
  • Designing logs first makes everything else—alerts, dashboards—trivial later.

🧠 Smart Product Categorization — Zero-Shot NLP

Automatic product→category validation using BART-MNLI plus lightweight rules.

Tech: Transformers (BART-MNLI), pipelines, pandas

Zero-Shot Categorization with BART-MNLI

Model
BART-MNLI
CPU throughput
~8/s
Flag rate
~12%

Goal: Catch subtle mislabels that keyword rules miss, without training data.

Approach
  1. Parse category path and take last node as candidate label.
  2. Quick keyword pass for obvious matches → early accept.
  3. Run zero-shot entailment on product name vs. label; threshold = 0.6.
  4. Low-score items → review queue with top-k alternative labels.
Evaluation
  • Manually labeled 200 items; ~0.92 precision on flagged set.
  • Common confusions: apparel variants (“top” vs “blouse”), bedding (“duvet” vs “comforter”).

Outcome: materially reduces manual QA while keeping reviewers focused on true edge cases.
